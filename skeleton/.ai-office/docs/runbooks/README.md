# Runbooks

Operational documents: plans, status files, review reports, and task breakdowns.

## Naming convention

- `<slug>-status.md` — pipeline state tracker
- `<slug>-plan.md` — project plan and milestones
- `<slug>-review.md` — multi-sector review document
- `<slug>-tasks.md` — task breakdown summary

## Create a runbook

```
/office-scaffold <slug> status
/office-scaffold <slug> plan
/office-scaffold <slug> review
```
